# Описание макроса

Макрос предназначен для экспорта **одного активного чертежа (.idw) в PDF** с автоматическим извлечением номера детали из iProperties и системой повторных попыток при ошибке записи файла.

Работает только с **открытым документом Inventor**.

---

## Принцип действия

### Основные сведения

Макрос отрабатывает по структуре:

```
КОРЕНЬ_РЕД_ФОРМАТ
|
├── 1_Архив
|
├── Пакет_1 
│   ├── файл1.idw (где запускаешь макрос)
│   ├── файл2.idw
│
├── Пакет_2
│   ├── файл3.idw
|
КОРЕНЬ_PDF_ФОРМАТ (где PDF раскладываются без подкаталога, по с именем, дублирующим имя файла)

```

Макрос работает по простой цепочке:

    - ОТКРЫТЫЙ ДОКУМЕНТ (Drawing)
    - Проверка типа документа
    - Извлечение Part Number из iProperties
    - Fallback на имя файла
    - Очистка строки (sanitizer)
    - Формирование пути 2_PDF
    - Подготовка PDF Translator
    - Настройка параметров экспорта
    - Попытка сохранения PDF (до 3 раз)
    - Вывод результата


### Алгоритм

1. Проверка документа

   - Получение активного документа Inventor
   - Проверка типа:
     - если не DrawingDocument → завершение макроса

2. Определение имени файла (Part Number)

   - Пытаемся получить:
     - `Design Tracking Properties → Part Number`
   - Если пусто или ошибка:
     - берём имя файла чертежа (без расширения) -> iProperties → если пусто → имя файла

3. Очистка имени файла

   - Удаление запрещённых символов файловой системы:
     - `/ \ : * ? " < > |`
   - Замена на `_`

4. Защита от
   
   - падений SaveCopyAs
   - битых путей
   - некорректных имен Windows

5. Формирование PDF структуры

   - Базовая папка определяется от текущего документа:
     - `..\..\2_PDF\`
   - Если папки нет:
     - создаётся автоматически
   - Финальный путь: 2_PDF/<PartNumber>.pdf

6. PDF Translator

   - Получение встроенного AddIn:
     - PDF Export Engine
   - Проверка наличия:
     - если отсутствует → остановка

7. Контекст экспорта

   - Создаётся TranslationContext
   - Режим:
     - FileBrowseIOMechanism

8. Настройка параметров PDF

    Если поддерживается AddIn:

       - Цвет:
       - не чёрный принудительно
       - Толщина линий:
       - без удаления
       - Разрешение:
       - 400 DPI
       - Листы:
       - все листы чертежа

9. DataMedium (путь сохранения)

   - Формируется объект сохранения
   - Указывает полный путь PDF

10. Механизм повторных попыток

    - попытка SaveCopyAs
    - если ошибка:
      - сохраняем exception
      - ждём 1 сек
      - повторяем

11. Завершение

    - если успех:
      - сообщение с путём PDF
    - если провал:
      - вывод ошибки + текст последнего исключения


## Ключевые особенности

* Экспорт одного активного чертежа
* Автоматический fallback имени (iProperty → файл)
* Защита от некорректных символов имени
* Retry-механизм (устойчивость к блокировкам файла)
* Работа без ручных настроек
* Полная автономность исполнения
