# Описание макроса

Макрос предназначен для пакетного экспорта PDF-файлов из всех `.idw` чертежей, находящихся в текущей рабочей папке, с формированием единой структуры вывода в каталог `2_PDF`.

Внутри каталога создаётся логическая подкатегория экспорта (например: *Обечайка*), в которую складываются все PDF-файлы.

---

## Принцип действия

### Основные сведения

Макрос отрабатывает по структуре:

```
КОРЕНЬ_РЕД_ФОРМАТ
├── 1_Архив
|
├── Пакет_1 
│   ├── файл1.idw (желательно сборка, изнутри которой запускается макрос)
│   ├── файл2.idw
│
├── Пакет_2 
│   ├── файл3.idw (желательно сборка, изнутри которой запускается макрос)
|
КОРЕНЬ_PDF_ФОРМАТ (где PDF раскладываются по подкаталогам, дублирующим имена папок из которых был произведен вызов)

```


---

## Алгоритм

1. Определение исходных путей

   - Получение пути текущей папки (`root_directory`)
   - Формирование базовой папки вывода:
     - `2_PDF`
   - Создание пользовательской подкатегории:
     - `subfolder_name` (задаётся вручную)

2. Формирование структуры вывода

    Создаётся иерархия:

            ```
            2_PDF/
            └── <subfolder_name>/
            ```

    Если папки не существуют — они создаются автоматически.


3. Получение списка чертежей

   - Сканируются все файлы в текущей папке
   - Фильтр:
     - `*.idw`

   Если файлов нет:
   - выполнение макроса завершается


4. Подготовка PDF Translator

   - Получение встроенного Autodesk PDF AddIn
   - Создание TranslationContext
   - Режим:
     - FileBrowseIOMechanism

5. Обработка файлов

    Для каждого `.idw` файла выполняются шаги:

    5.1 Открытие документа
        - Чертёж открывается из текущего каталога


    5.2 Получение имени файла

        ```
        iProperties → Design Tracking Properties → Part Number
        ```

        Fallback:

        ```

    5.3 Sanitizer имени

    Выполняется очистка строки:

    - удаляются запрещённые символы файловой системы:
        - `/ \ : * ? " < > |`
    - заменяются на `_`

    предотвращает:
    - ошибки сохранения PDF
    - некорректные пути Windows
    - сбои SaveCopyAs


    5.4 Формирование пути PDF. Итоговый путь:

    ```
    2_PDF/<subfolder_name>/<PartNumber>.pdf
    ```

    5.5 Настройка PDF экспорта

        Если поддерживается AddIn:

          - Цвет: стандартный
          - Линии: без упрощения
          - Разрешение: 400 DPI
          - Листы: все листы чертежа

    5.6 Экспорт PDF

    - Создаётся DataMedium
    - Выполняется `SaveCopyAs`

    5.7 Счётчик успешных операций

    - При успешном экспорте увеличивается `success_counter`

6. Обработка ошибок

   - Ошибка на файл:
     - отображается сообщение
     - выполнение продолжается

   - Документ всегда закрывается в блоке `Finally`

7. Завершение работы

    После завершения обработки:

    - выводится статистика:
        - успешно экспортированные файлы / общее количество
    - автоматически открывается папка результата:

```
explorer.exe → 2_PDF/<subfolder_name>
```


## Ключевые особенности

- Экспорт всех `.idw` из одной директории
- Гибкая логическая структура через `subfolder_name`
- Автоматический fallback имени (iProperties → filename)
- Sanitizer для безопасных имён файлов
- Полностью автономная работа без ручной настройки
- Автоматическое открытие результата
- Подсчёт успешных экспортов
