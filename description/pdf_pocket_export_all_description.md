# Описание макроса

Макрос предназначен для пакетного экспорта PDF-файлов всех чертежей, размещённых в подпапках папки вызова (кроме папки 1_Архив) с глубиной вложенности 1 уровень:

**КОРЕНЬ → ПАПКА ПАКЕТА ЧЕРТЕЖЕЙ**

Макрос обходит подпапки, определяет наличие чертежей .idw, и для каждой папки с чертежами формирует соответствующую структуру в папке 2_PDF, дублируя имена каталогов и файлов.


## Принцип действия

### Основные сведения

Макрос отрабатывает по структуре:

```
КОРЕНЬ_РЕД_ФОРМАТ
|
├── pocket_export.idw (запускать макрос тут)
|
├── 1_Архив   ← игнорим
|
├── Пакет_1
│   ├── файл1.idw
│   ├── файл2.idw
│
├── Пакет_2
│   ├── файл3.idw
|
КОРЕНЬ_PDF_ФОРМАТ (где PDF раскладываются по подкаталогам, дублирующим структуру РФ)

```

### Алгоритм

**Алгоритм работы макроса экспорта PDF**


1. Определение исходных путей
   
   - Получение пути папки запуска макроса (rootDir)
   - Формирование пути к корневой папке для PDF (2_PDF) относительно rootDir
   - Создание папки 2_PDF, если она отсутствует
  
2. Получение списка подпапок
   
   - Чтение всех подпапок в rootDir
   - Исключение служебной папки 1_Архив из обработки
  
3. Обнаружение пакетов чертежей. Для каждой подпапки:

   - Получение списка файлов .idw
   - Проверка наличия чертежей:
     - Если файлов нет → папка пропускается
     - Если есть → папка считается пакетом чертежей
  
4. Формирование структуры выходных каталогов. Для каждой найденной папки с чертежами:
   
   - Формируется путь в 2_PDF, повторяющий имя исходной папки
   - Создаётся соответствующая подпапка, если она отсутствует

5. Подготовка к экспорту PDF
   
   - Получение PDF Translator AddIn
   - Создание контекста трансляции (TranslationContext)
   - Настройка параметров экспорта (цвет, разрешение, диапазон листов)

6. Обработка чертежей

  - Для каждого .idw файла в пакете:
  - Открытие документа
  - Открытие чертежа в невидимом режиме
  - Формирование имени PDF по имени файла
  - Очистка строки от недопустимых символов файловой системы
  
7. Формирование пути PDF

   - Генерация полного пути: 2_PDF/<Имя_папки>/<Имя_файла>.pdf
   - Экспорт
   - Создание DataMedium
   - Вызов SaveCopyAs для сохранения PDF
  
8. Завершение обработки файла
   
   - Закрытие документа
   - Обработка возможных ошибок
  
9.  Завершение работы
    
    - Подсчёт общего количества успешно экспортированных файлов
    - Вывод итогового сообщения
    - Открытие папки 2_PDF в проводнике
  
### Ключевые особенности алгоритма

* Автоматическое определение пакетов (по наличию .idw)
* Зеркалирование структуры каталогов
* Безопасная генерация имён файлов
* Пакетная обработка без ручной настройки
* Игнорирование служебных папок
